# LinkedIn Post

Today I completed a cloud security investigation lab focused on Azure attack path analysis.

The scenario involved a publicly exposed Azure VM with a critical vulnerability, an assigned Managed Identity, and access to a Storage Account containing potentially sensitive data.

The key lesson:

Cloud security risk is not always about one single misconfiguration. The real risk often appears when multiple issues connect together — public exposure, vulnerability, identity permissions, and sensitive data access.

This lab helped me practise:

- Azure attack path analysis
- Public exposure review
- Vulnerability risk assessment
- Managed Identity and RBAC review
- Storage Account access risk
- Consultant-style documentation using Finding, Evidence, Risk, Recommendation, and Priority
